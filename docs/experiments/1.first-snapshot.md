# Experiment: Creating and Restoring a MicroVM Snapshot

## Objective

Validate that a Firecracker microVM can be snapshotted and restored successfully.
This experiment demonstrates the core primitive required by SnapVM: the ability to
capture the complete state of an execution environment and restore it later.

## Background

Snapshots allow a virtual machine to save its entire state, including:

- memory (RAM)
- CPU state
- device state
- filesystem state

When restored, the VM should continue from the exact point in time where the snapshot was created.

This capability enables fast rollback, reproducibility, and safe experimentation.

## Requirements

System requirements:

- Linux machine (Ubuntu recommended)
- KVM enabled
- Firecracker installed
- `curl` for interacting with the Firecracker API

Required files:

- Linux kernel image (example: `vmlinux.bin`)
- root filesystem image (example: `rootfs.ext4`)

Verify KVM support:

```bash
ls /dev/kvm
```

If the device exists, virtualization is available.

## Setup

Start Firecracker:

```bash
firecracker --api-sock /tmp/firecracker.socket
```

The Firecracker API will now listen on:

```
/tmp/firecracker.socket
```

## Steps

1. Configure the VM.

Example machine configuration:

```bash
curl --unix-socket /tmp/firecracker.socket -i -X PUT 'http://localhost/machine-config' -H 'Accept: application/json' -H 'Content-Type: application/json' -d '{
  "vcpu_count": 1,
  "mem_size_mib": 512
}'
```

2. Attach the root filesystem.

3. Load the kernel.

4. Boot the VM.

5. Once the VM is running, trigger snapshot creation using the Firecracker snapshot API.

Example snapshot files:

```
snapshot.snap
memory.snap
```

6. Stop the VM.

7. Restore the VM using the snapshot files.

## Expected Result

The VM should restart from the exact state stored in the snapshot.
All running processes and memory state should match the moment the snapshot was taken.

## Validation

The experiment is considered successful if:

- the VM boots from the snapshot
- processes continue execution normally
- the filesystem state is preserved

## Notes

This experiment validates the fundamental capability needed for SnapVM:
the ability to **persist and restore execution environments instantly**.
