# Experiment: Parallel MicroVM Execution

## Objective

Demonstrate that multiple microVM instances can be launched from the same base snapshot.

This experiment explores how snapshot cloning can support parallel environment execution.

## Background

When multiple environments share the same snapshot as a starting point,
the system can avoid rebuilding identical environments repeatedly.

Using techniques like **Copy-on-Write (CoW)** memory management,
multiple virtual machines can share most of their memory until modifications occur.

This enables efficient parallel execution.

## Requirements

- Firecracker installed
- base snapshot created from a configured VM
- sufficient system memory

Example snapshot files:

```
snapshot.snap
memory.snap
```

## Setup

Create a base environment inside a microVM.

Install any necessary tools or dependencies.

Example:

- install Python
- install Node.js
- configure application environment

Once the environment is prepared, create a snapshot.

This snapshot will be used as the base for all parallel VMs.

## Steps

1. Create a base snapshot of the configured VM.

2. Launch multiple microVM instances using the same snapshot.

Example conceptual structure:

```
VM-A ← snapshot
VM-B ← snapshot
VM-C ← snapshot
```

3. Execute different commands inside each VM.

Examples:

VM-A:
```
run test suite
```

VM-B:
```
run build process
```

VM-C:
```
run application server
```

4. Observe system behavior and resource usage.

## Expected Result

Each VM should:

- start from the same initial environment
- operate independently
- maintain isolation from the other VMs

Memory usage should remain efficient due to shared snapshot state.

## Validation

The experiment is successful if:

- multiple VMs start correctly from the snapshot
- environments diverge independently
- system resources remain manageable

## Notes

Parallel VM execution is important for workloads that explore multiple execution paths,
such as automated testing systems or AI agent exploration.
